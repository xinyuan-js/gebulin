# 路线图

## 第 0 阶段：项目立项

目标：明确项目目的和基本规范。

- 建立 README。
- 建立项目章程。
- 建立架构原则。
- 建立开发规范。
- 建立目录约定。
- 确认基础技术栈。

完成标准：

- 新成员能在 10 分钟内理解项目方向。
- 后续实现有明确技术边界。

## 第 1 阶段：工程骨架

目标：建立可长期维护的基础工程。

基础技术栈：

- Vue + Nuxt。
- Go + Gin + Gorm，拆分 OAuth 服务和哥布林主业务服务。
- PostgreSQL + Redis + Kafka。
- Nginx + Docker。

需要完成：

- 完善 Dev Container，保证打开仓库即可进入统一开发环境。
- 创建 `apps/web` Nuxt 应用。
- 创建 `services/oauth` OAuth 服务。
- 创建 `services/goblin` 哥布林主业务服务。
- 建立开发容器内的一键启动流程。
- 接入 PostgreSQL、Redis、Kafka、Nginx。
- 增加 `.env.example`。
- 提供 Web、OAuth、哥布林主业务健康检查。

完成标准：

- 可以通过 Dev Container 运行本地开发环境。
- 有最小 CI 或本地检查命令。
- 有 `.env.example`。

## 第 2 阶段：最小可用巢穴

目标：做出第一个可访问版本。

建议范围：

- 像素风 RPG 基地首页。
- 场景热点入口和基础导航。
- 巢穴基地地图雏形。
- 一个内容列表或公告区。
- 基础错误页和空状态。

完成标准：

- 用户打开站点能理解主题。
- 用户能从基地场景点击进入至少 2 个模块。
- 开发者能按模块继续添加新功能。

## v0 业务模块范围

v0 的目标是让当前地图上出现过的每个模块都有可访问页面、清晰职责和最小业务界面。具体范围如下：

- `公告板`：公告、更新日志、活动入口。
- `篝火 / 大篝火`：广场、闲聊、动态聚集点。
- `洞穴深处`：进入洞穴第一层的地图入口。
- `铺盖卷`：个人管理中心，包含身份牌、收藏、账号相关入口。
- `仓库`：收藏、道具、库存记录。
- `石碑`：称号、成就、排名。
- `哥布林帐篷`：管理系统入口，后续按哥布林身份分级。
- `哥布林守卫`：登录和身份校验入口，对接独立 OAuth。
- `女神石像`：未来智能体入口，按哥布林身份提供不同工具或子智能体能力。

OAuth 与哥布林身份的 v0 数据范围：

- OAuth 账号持有邮箱、密码哈希、账号状态和登录审计。
- 哥布林档案持有头像、哥布林名、金币、哥布林 UID 和身份等级。
- 哥布林 UID 从 `10001` 起。
- UID `10001` 自动成为 `哥布林之神`，作为内置超级管理员身份；v0 权限能力可以先等同 `哥布林长老`。
- 账号注册后必须自动创建哥布林档案。

v0 完成标准：

- 每个模块至少有一个 Nuxt 页面。
- 每个模块在地图中有明确入口或归属层级。
- 未接后端的模块也要有前端业务骨架，不能只显示一句占位文案。
- 后续接 Go 服务时，页面结构不需要大改。

## 第 3 阶段：用户和内容系统

目标：形成基础社区能力。

建议范围：

- 注册、登录、退出。
- 用户资料。
- 内容发布。
- 评论。
- 标签或分区。
- 基础审核和举报。

完成标准：

- 权限校验不依赖前端。
- 登录和 Token 签发归 OAuth 服务，业务资料归哥布林主业务服务。
- 核心发布链路有测试。
- 用户生成内容有治理入口。

## 第 4 阶段：巢穴扩展

目标：让项目从网站变成平台。

可能方向：

- 隐藏入口。
- 称号和成就。
- 积分或声望。
- 任务系统。
- 活动页。
- 小游戏。
- 后台管理。

完成标准：

- 新模块可以独立接入。
- 不需要改动大量旧代码才能上线新功能。
- 关键行为有日志、监控或审计能力。
