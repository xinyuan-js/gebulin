# 项目章程

## 项目名称

哥布林巢穴。

## 项目目的

哥布林巢穴是一个以“哥布林聚集地”为核心隐喻的 Web 平台。它强调阴暗、混乱、低成本生存感、黑色幽默和反主流气质。项目可以从一个小型网站开始，但设计上必须允许未来不断长出新的入口、频道、系统和玩法。

一句话目标：

> 建立一个可扩展的负能量风格 Web 巢穴，让内容、互动和功能模块都能围绕“哥布林聚集”持续生长。

## 产品气质

- 阴暗但可读：视觉和文案可以压抑、脏乱、尖锐，但不能牺牲可用性。
- 混乱但有规则：用户感受到巢穴四通八达，开发者仍能找到清晰模块边界。
- 讽刺但不越界：允许黑色幽默、失败叙事、自嘲和荒诞表达；避免现实人身攻击、仇恨煽动、违法引导和直接伤害鼓励。
- 低信任环境：产品设计默认用户会尝试钻空子，因此权限、审计、限流和内容治理要尽早进入架构考虑。

## 主业务形态

哥布林主业务首页应是一个像素风 RPG 基地，而不是普通门户或信息流。用户进入首页时，应看到一个空旷、阴暗、可探索的基地场景；场景中的公告板、篝火、洞穴、仓库、石碑等地点可以点击，并跳转到对应业务模块。

首页的核心价值是建立世界观和模块入口：

- 空旷：保留基地空间感，不用内容卡片填满首屏。
- 可点：场景中的地点就是功能入口。
- 可扩展：未来新增模块时，可以像在基地里增加新地点一样接入。
- 可识别：像素风不能妨碍用户理解入口和状态。

## 目标用户

早期目标用户可以是：

- 喜欢黑色幽默、怪诞社区、地下风格内容的人。
- 喜欢探索隐藏页面、分支路径和成就收集的人。
- 喜欢参与共同搭建设定、内容和玩法的人。

## 长期形态

项目未来可能扩展为：

- 内容社区：帖子、评论、收藏、标签、专题。
- 巢穴地图：像素风基地、分区入口、隐藏路径、权限门槛。
- 用户身份：称号、等级、声望、库存、成就。
- 小游戏或互动工具：抽卡、任务、投票、排行榜。
- 运营后台：内容审核、用户管理、配置开关、活动管理。
- API 平台：为 Web、移动端或外部工具提供统一能力。

## 项目边界

必须做：

- 保持项目气质统一。
- 保持模块可插拔、可替换。
- 保持文档随实现更新。
- 保持核心路径可测试。

不做：

- 为短期页面效果牺牲长期结构。
- 在没有需求证据时引入复杂基础设施。
- 把风格上的“负能量”变成现实伤害、骚扰或违法内容。
- 用全局共享状态、硬编码路由和散乱脚本堆出不可维护系统。
